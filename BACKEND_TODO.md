## 💡 Suggestions & Optimizations for "Low Cost / Free"
1. **Leaderboard Read Costs**: Querying thousands of users to find the top 50 every time someone opens the leaderboard will exhaust Firestore's free reads fast. **Solution**: A backend job calculates the Top 50 once a day and writes it to a single `leaderboards/daily` document. Clients only do 1 read.
2. **Cold Starts on Render**: Render's free web services spin down after 15 minutes of inactivity. **Solution**: Have the frontend show a smooth "Waking up server..." loading state, or use a free cron service (like cron-job.org) to ping the API every 14 minutes.
3. **Chat Reads**: For global chat, Firestore document reads can add up. Consider capping the chat history loaded on the frontend to the last 20 messages, and aggressively cleaning up messages older than 24 hours via a backend script.
4. **Player Model Updates**: Extend the existing `PlayerModel` to include:
   - `role` (e.g., `admin`, `mod`, `player`).
   - `muteUntil` (timestamp for chat muting).
